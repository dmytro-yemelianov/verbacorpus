# verba — SP12: UI internationalization (10 languages, URL-prefixed)

**Date:** 2026-06-24
**Status:** Approved (design)
**Sub-project:** 12 — translate the **UI chrome** (not the corpus) into 10 languages, served at URL-prefixed routes with per-language SEO.
**Repo:** `dmytro-yemelianov/verbacorpus`; `app/` (Worker + PWA + pages).
**Depends on:** the live app (SP4–SP11): the Worker (`index.ts`), PWA (`main.ts`), `/p/:id` pages (`meta.ts`), `about.html`, `api.html`, the 27-theme taxonomy.

---

## 1. Scope

Localize everything a human **reads in the interface** into 10 languages; the **corpus content stays Ukrainian**. URL-prefixed routes (`/de/…`) with server-rendered per-language `<head>` for real SEO; `/` (no prefix) is the canonical Ukrainian site (no auto-redirect).

**Translated (UI chrome):** masthead eyebrow + standfirst, search placeholder, filter labels, the **27 category display-labels**, every button/toggle/state (search · «за змістом» · «Гортати» · «♥ Збережені» · «Показати ще» · «Поділитися»/«Картка» · «Інше прислів'я» · «Закрити» · empty states · «Схожі прислів'я»/«Варіанти»), the colophon copy, the **About page** prose, the **`/p/:id`** page chrome, **OG/Twitter meta** descriptions, and the **`api.html`** developer docs.

**Never translated:** proverb `text`, `modern_text`, `explanation`, source proper-names, catalog/category *keys*, the `/card/:id.png` images (the proverb is the content), `llms.txt`, the JSON data, the API response bodies.

**Languages (10):** `uk` (base/source) · `en` · `de` · `fr` · `es` · `pl` · `it` · `pt` · `ja` · `zh` (Simplified). Russian deliberately excluded.

## 2. Architecture

**Message catalogs** — `app/public/i18n/{lang}.json`, flat `key → string` maps. **`uk.json` is the source-of-truth**; every other catalog has the same keys. The 27 category labels live under `cat.<key>` (e.g. `cat.work_labor`). Catalogs are both **served as assets** (client fetch) and **bundled into the Worker** (esbuild JSON import) for server-side rendering.

**Pure helper** — `app/src/shared/i18n.ts`:
- `LANGS = ["uk","en","de","fr","es","pl","it","pt","ja","zh"]`, `DEFAULT_LANG = "uk"`.
- `parseLang(pathname) -> { lang, rest }` — strips a leading `/<lang>/` prefix (or `/<lang>` exact); no prefix → `{ lang: "uk", rest: pathname }`.
- `t(catalog, key, fallback?) -> string` — catalog lookup with uk-catalog fallback then the key.
- `hreflangLinks(pathWithoutLang, host) -> string` — `<link rel="alternate" hreflang="<lang>" href="https://host/<lang>/…">` for all 10 + `x-default` (→ uk/root).

**Client** (PWA): the Worker injects `<script>window.__LANG__="de"</script>`; `main.ts` loads `/i18n/<lang>.json`, exposes `t(key)`, applies it to **dynamically-rendered** strings (results head, swipe, saved, counts) and category labels (`catLabel(k) = t("cat."+k)`). Static chrome is translated **server-side** (below) to avoid a flash.

**Worker** (`index.ts`) becomes the front door for **HTML pages** (static assets — `/styles.css`, `/app.js`, `/data/*`, `/fonts/*`, `/card/*`, `/i18n/*` — still pass straight to ASSETS):
- Parse the lang prefix. `/uk/…` → **301** to the unprefixed path (canonical). `/` and `/<lang>/` → the SPA; `/<lang>/about` → About; `/<lang>/api.html` → api docs; `/<lang>/p/:id` → proverb page.
- For SPA / About / api.html: fetch the base HTML from ASSETS and run **HTMLRewriter** to (a) set `<html lang="de">`; (b) replace the text of every `[data-i18n="key"]` element with `t(catalog, key)`; (c) replace `[data-i18n-attr="meta:content:key"]`-style head metas (title, description, og:title/description) with the lang string; (d) inject `<script>window.__LANG__</script>` + `hreflangLinks(...)` + a per-language `<link rel="canonical">`. The body's dynamic parts remain client-rendered via the catalog.
- For `/<lang>/p/:id`: `buildProverbPage(proverb, host, catalog)` — chrome + OG meta in the language, **proverb stays Ukrainian**, OG image = the unprefixed `/card/:id.png`, hreflang alternates emitted.

**Routing config** — `wrangler.jsonc` `run_worker_first` adds `"/"`, `"/<lang>/*"` for each lang, `"/about"`, `"/api.html"` (alongside the existing `/api/*`,`/p/*`,`/card/*`). Static asset paths are not listed, so they bypass the Worker.

## 3. Surfaces & string extraction

- **`index.html`, `about.html`, `api.html`** — add `data-i18n="key"` to each translatable text node (keep the existing Ukrainian as the in-file default = the uk source strings); head metas marked for translation. The Worker translates them per request; uk needs no rewrite (defaults already uk).
- **`main.ts`** — dynamic strings → `t(key)`; `catLabel` → `t("cat."+key)`; the language switcher.
- **`meta.ts`** `buildProverbPage` / `cardModel` — gains the catalog; chrome/OG translated, proverb untouched (cards unchanged — language-neutral).
- The 27 `cat.*` strings come from the current `meta.taxonomy` Ukrainian labels (seed `uk.json`); the API `/api/v1/categories` keeps returning the uk labels (data, out of scope for per-request lang).

## 4. Language switcher

A compact menu in the masthead (e.g. a `<details>` or `<select>` styled to the theme) listing the 10 languages by endonym (Українська, English, Deutsch, Français, Español, Polski, Italiano, Português, 日本語, 中文). Selecting one navigates to `/<lang>/<current-rest>` (preserving the path, e.g. `/de/p/p000001`), persisted in `localStorage`. On `/` (uk), a one-time subtle hint may suggest the Accept-Language match (no forced redirect). Keyboard-accessible.

## 5. SEO

Every page emits, server-side: `<html lang>`, a per-language `<link rel="canonical">`, and `<link rel="alternate" hreflang="<lang>">` for all 10 + `x-default` (→ `/`). Optionally a future sitemap lists the language variants (out of scope now). Per-language `og:locale`.

## 6. Translation production (controller phase)

Build the engine and ship **uk + en** first (proves the full pipeline end-to-end). Then a controller phase LLM-batch-translates `uk.json` → the other 8 (`de fr es pl it pt ja zh`), each reviewed (a second-pass or spot check) for tone/accuracy; a **completeness check** asserts every catalog has exactly the `uk.json` key set. Translations cover the UI strings + the 27 category labels + the About/api prose keys. Quality caveat: machine-assisted translations, reviewed; corrections welcome via GitHub.

## 10. Risks / open items

- **Worker becomes the HTML front door** — the biggest change. Static assets must keep bypassing it (explicit non-listing in `run_worker_first`); the `/api/*` chain must stay intact. Verified by the routing tests.
- **HTMLRewriter translation pass** — server-translating `[data-i18n]` avoids FOUC and gives SEO-complete pages, but adds per-request CPU; cheap (small DOM) and the result is edge-cacheable. uk needs no rewrite (fast path).
- **CJK rendering** — ja/zh chrome relies on the system-ui font fallback (the proverbs stay Spectral/Ukrainian); verify on preview. No bundled CJK font.
- **FOUC / flash** — eliminated for static chrome by server-side translation; the dynamic (client-rendered) strings appear in the active language because the catalog is loaded before first render (gate on catalog load, with uk fallback if it fails).
- **Catalog completeness / drift** — the completeness test gates merges; `uk.json` is the single source of truth, new keys added there first.
- **Translation quality** — machine-assisted + reviewed; long-form (About/api) is the riskiest — flag for a closer review; the corpus content is unaffected so a chrome mistranslation is low-stakes and fixable.
- **SW + per-lang URLs** — each `/<lang>/` URL caches independently; ensure the SW doesn't serve the uk shell for a localized URL (cache by full URL).
- **API `/categories` language** — stays Ukrainian (data); a `?lang=` on the API is out of scope (future).
